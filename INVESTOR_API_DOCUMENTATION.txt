ANUSHA TRADE - INVESTOR API DOCUMENTATION FOR FRONTEND TEAM
============================================================

Backend base URL:
- Local: http://localhost:8080
- Production example: https://api.anushatrade.com

Frontend env:
- VITE_API_BASE_URL=http://localhost:8080
- In production set VITE_API_BASE_URL=https://api.anushatrade.com

Content types:
- JSON APIs: Content-Type: application/json
- File upload APIs: multipart/form-data

Authentication:
- Protected investor APIs require:
  Authorization: Bearer <accessToken>
- accessToken is returned by login/register onboarding APIs.
- refreshToken is returned by login/register onboarding APIs and can be used to get a new accessToken.

Common error response:
{
  "timestamp": "2026-05-28T10:30:00",
  "status": 400,
  "error": "Bad Request",
  "message": "Validation failed",
  "path": "/api/kyc/submit",
  "details": ["field: error message"]
}

Important enums used by frontend:
- Role: INVESTOR, ADMIN, SUPER_ADMIN
- KycStatus: NOT_SUBMITTED, PENDING, APPROVED, REJECTED, REUPLOAD_REQUIRED
- AccountStatus: PENDING, ACTIVE, SUSPENDED, DEACTIVATED
- OnboardingStatus: REGISTERED, KYC_PENDING, KYC_COMPLETED, BANK_LINKED, ACCOUNT_ACTIVATED, ACTIVE
- InvestmentStatus: PENDING_RECEIPT, RECEIPT_UPLOADED, ACTIVE, MATURED, CANCELLED, EARLY_WITHDRAWAL, REJECTED
- PaymentMode: NEFT, RTGS, IMPS, UPI, CASH, CARD, NETBANKING, WALLET
- WithdrawalStatus: PENDING, APPROVED, REJECTED, PROCESSED


INVESTOR FRONTEND FLOW
======================

Recommended screen flow:

1. Send OTP or Firebase phone OTP
2. Verify OTP
3. Register investor
4. Login and store tokens
5. Submit KYC documents
6. Poll/check KYC status
7. After admin approves KYC, link bank
8. Activate account
9. Set MPIN
10. Open dashboard
11. Show plans
12. Create investment
13. Pay using Razorpay OR upload manual receipt
14. Track investments, wallet, withdrawals, notifications

Frontend routing suggestion based on status:
- Missing auth token: /login
- kycStatus NOT_SUBMITTED or empty: /kyc
- kycStatus PENDING: /kyc/status
- kycStatus REUPLOAD_REQUIRED or REJECTED: /kyc
- kycStatus APPROVED and bankVerified false: /bank-link
- bankVerified true and accountStatus PENDING: /activate
- accountStatus ACTIVE and mpinCreated false: /setup-mpin
- accountStatus ACTIVE: /dashboard


JAVASCRIPT FETCH HELPERS
========================

const BASE_URL = import.meta.env.VITE_API_BASE_URL || "http://localhost:8080";

async function apiJson(path, { method = "GET", body, token } = {}) {
  const headers = { "Content-Type": "application/json" };
  if (token) headers.Authorization = `Bearer ${token}`;

  const res = await fetch(`${BASE_URL}${path}`, {
    method,
    headers,
    body: body ? JSON.stringify(body) : undefined,
  });

  const data = await res.json().catch(() => ({}));
  if (!res.ok) throw new Error(data.message || `Request failed ${res.status}`);
  return data;
}

async function apiForm(path, formData, token) {
  const headers = {};
  if (token) headers.Authorization = `Bearer ${token}`;

  const res = await fetch(`${BASE_URL}${path}`, {
    method: "POST",
    headers,
    body: formData,
  });

  const data = await res.json().catch(() => ({}));
  if (!res.ok) throw new Error(data.message || `Request failed ${res.status}`);
  return data;
}

15. Submit Full KYC
-------------------
POST /api/kyc/submit
Protected
Content-Type: multipart/form-data

Multipart fields:
- panCardImage: file, required, jpg/jpeg/png/pdf
- aadhaarFrontImage: file, required, jpg/jpeg/png/pdf
- aadhaarBackImage: file, required, jpg/jpeg/png/pdf
- selfiePhoto: file, required, jpg/jpeg/png
- bankPassbookOrStatement: file, required, jpg/jpeg/png/pdf
- panNumber: string, required if not already saved on user
- aadhaarLast4: string, required if not already saved on user, 4 digits
- dateOfBirth: string, required if not already saved on user, yyyy-mm-dd
- address: string, required if not already saved on user

Profile photo aliases supported by backend:
- selfiePhoto
- profilePhoto
- PROFILE_PHOTO

FRONTEND IMPLEMENTATION NOTES
=============================

1. Never set Content-Type manually for FormData.
   Browser will set multipart boundary automatically.

2. Always include Authorization for protected APIs:
   Authorization: Bearer <accessToken>

3. On 401/403:
   - Try /api/auth/refresh-token using refreshToken.
   - If refresh fails, clear auth storage and go to /login.

4. KYC upload file names must match backend exactly.
   Full KYC required fields:
   - panCardImage
   - aadhaarFrontImage
   - aadhaarBackImage
   - selfiePhoto OR profilePhoto OR PROFILE_PHOTO
   - bankPassbookOrStatement

5. KYC backend now supports S3 fallback to local storage if S3 credentials/bucket are not configured.
   This prevents S3 failure from skipping DB save.

6. Sequential upload UI should not crash on first failed document.
   Recommended behavior:
   - Validate all required files before uploading.
   - Prefer one full /api/kyc/submit request.
   - If using separate APIs, collect per-document errors and show user which document failed.

7. For Razorpay:
   - Use /api/payments/razorpay/checkout-order first.
   - Open Razorpay Checkout using checkout.orderId and checkout.keyId.
   - In handler, call /api/payments/razorpay/verify.
   - Only show investment as active after verify succeeds.

8. For manual payment:
   - Call /api/investments/apply.
   - Then upload receipt using /api/investments/{investmentId}/upload-receipt.
   - Status becomes RECEIPT_UPLOADED until admin approval.

9. For document preview:
   - Use /api/files/view?path=<encoded path>.

10. Store these fields from login responses:
   - accessToken
   - refreshToken
   - role
   - userId
   - kycStatus
   - accountStatus
   - onboardingStatus
   - bankVerified
   - mpinCreated
